# Swagger generated server

## Overview
This server was generated by the [swagger-codegen](https://github.com/swagger-api/swagger-codegen) project.  By using the [OpenAPI-Spec](https://github.com/OAI/OpenAPI-Specification) from a remote server, you can easily generate a server stub.

### Using ngrok for AWS Lambda
Turn on the terminal and use instruction "ngrok http 6777" to generate public network of BE, it can be like this "https://xxxxxx.ngrok-free.app -> http://localhost:6777" (The left one is the public one)

Copy the public one into lambda configuration "BFF_INTERNAL_URL"

Then it will trigger to insert user into table "users" once the user sign up successfully

### Running the server
To run the server, run:

first create a ".env" file project root directory with the required contents

see https://medium.com/the-node-js-collection/making-your-node-js-work-everywhere-with-environment-variables-2da8cdf6e786
 for explanation 

first only run `npm install`

every time you want start the server run `npm run start_local`

If you want run in debug mode create the following VSCode launch file
```
{
    // Use IntelliSense to learn about possible attributes.
    // Hover to view descriptions of existing attributes.
    // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Launch BE via NPM",
            "request": "launch",
            "skipFiles": [
                "<node_internals>/**"
            ],
            "type": "node",
            "cwd":"${workspaceFolder}",
            "program": "${workspaceFolder}/index.js",
            "runtimeExecutable": "npm",
            "runtimeArgs": ["run-script", "start_local"]
        }
    ]
}
```

This project leverages the mega-awesome [swagger-tools](https://github.com/apigee-127/swagger-tools) middleware which does most all the work.
